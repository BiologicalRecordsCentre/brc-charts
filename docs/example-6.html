Using the BRC Charts Library

Yearly comparison charts

Include the required Javascript libraries and CSS in the page. We need the BRC Charts JS libary, the associated CSS and the D3 library which is an external external dependency of the BRC Charts library. In this example they are all included from CDNs. 

<script src="https://d3js.org/d3.v5.min.js"></script>
<script src="https://cdn.jsdelivr.net/gh/biologicalrecordscentre/brc-charts/dist/brccharts.umd.js"></script>
<link rel="stylesheet" type="text/css" href="https://cdn.jsdelivr.net/gh/biologicalrecordscentre/brc-charts/dist/brccharts.umd.css"></script>

All the examples on this page use data derived from records downloaded from the NBN in March 2021 from these datasets: Ladybird Survey of the UK and UK Ladybird Survey data from iRecord.

For the yearly chart, the data must be passed to the chart as an array of objects, each of which has these properties: taxon - the name of the taxon; year - four digit number indicating the year; one or more properties that express counts, e.g. number of records (or abundance,) of the given taxon in the given year. For example: 

[
  {
    taxon: 'Adalia bipunctata',
    year: 1992,
    dr736: 196,
    dr695: 43
  },
  {
    taxon: 'Adalia bipunctata',
    year: 1993,
    dr736: 251,
    dr695: 50
  },
  ...
]

In this example there are two properies for counts ('dr736' and 'dr695'), each representing a different source dataset.

The calling code is responsible for building this data structure. On this page all the data are stored in a CSV file with four columns (taxon, year, dr736, dr695) and the D3 function d3.csv is used to read it into an array of objects like that shown above before sending it to the yearly function. (See the page source if you want to see how that code looks.)

First we create a div element which will contain the chart generated by the yearly function. 

<div id="chart1"></div>

In the page's Javascript we call the brccharts.yearly method, passing in the data we want to display and other configuration options. The first example below uses a minimal configuration. The data object is an array of objects formatted as previously described. 

brccharts.yearly({selector: '#chart1', data: data, taxa: ['Anatis ocellata'], metrics: [{ prop: 'dr736' }]})

The taxa option specifies an array identifying the taxa in the dataset for which graphs are to be generated. The metrics is an array of objects, each of which specifies a plot within the graph. Commonly there will only be a single plot within each graph, but there can be any number of them. The metric objects specify the property within the input data that holds the data for the plot plus extra information to indicate how the plot is to be styled.

By default, the chart configures the y axis to cover the range of years represented in the data. Since many data sets include a few very early records (as here), this is rarely the desired range. In the next graph, the range has been specified as 1980 to 2020 by setting the minYear and maxYear options. In addition, a second taxon has been displayed by adding its name to the taxa array, resulting in two graphs. Furthermore, a second plot has been added to each graph. In this example each plot represents records drawn from different sources. The source datasets are identified in the legend (which is itself configured in the metrics array property). The interactivity option specifies which type, if any, of mouse event initiates interactive highlighting of plots. When set to 'mousemove' plots within a chart will be highlighted when the mouse pointer moves over either the legend entities or the plots themselves. 

brccharts.yearly({
  selector: '#chart2',
  taxa: ['Anatis ocellata', 'Harmonia axyridis'],
  metrics: [
    { prop: 'dr736', label: 'iRecord ', colour: 'red'},
    { prop: 'dr695', label: 'Ladybird survey of UK', colour: '#4188A3' }
  ],
  minYear: 1980,
  maxYear: 2020,
  showLegend: true,
  interactivity: 'mousemove'
})

In the next graph, the main configuration option to change is showCounts which has been added and set to 'line' this changes the plot from a bar chart to a line graph. (The default value for the showCounts option is 'bar', so if you don't specify it you get a bar chart.) Also added in the axisLeftLabel option to add a label to the y axis. The interactivity option is set to 'mouseclick' so that the interactive display of graphics only happens on a mouse click as opposed to mouse movement. 

brccharts.yearly({
  selector: '#chart2a',
  taxa: ['Anatis ocellata', 'Harmonia axyridis'],
  metrics: [
    { prop: 'dr736', label: 'iRecord ', colour: 'red'},
    { prop: 'dr695', label: 'Ladybird survey of UK', colour: '#4188A3' }
  ],
  minYear: 1980,
  maxYear: 2020,
  showCounts: 'line',
  margin: {left: 50, right: 50, bottom: 20, top: 20},
  axisLeftLabel: 'Number of records (bars)',
  showLegend: true,
  interactivity: 'mouseclick'
})

In the next example, the plot style has been changed back to bars and the number of taxa in the taxa array increased.

The following example demonstrates some other configuration options including those for specifying a chart title, sub-title and footer. 

brccharts.yearly({
  selector: '#chart4',
  taxa:['Harmonia axyridis', 'Harmonia axyridis form conspicua', 'Harmonia axyridis form spectabilis', 'Harmonia axyridis form succinea'],
  metrics: [
    { prop: 'dr736', label: 'iRecord ', colour: 'red' },
    { prop: 'dr695', label: 'Ladybird survey of UK', colour: '#4188A3' }
  ],
  minYear: 2004,
  maxYear: 2020,
  perRow: 2,
  width: 350,
  height: 250,
  margin: {left: 40, right: 40, bottom: 20, top: 20},
  taxonLabelItalics: true,
  taxonLabelFontSize: 12,
  showLegend: true,
  interactivity: 'mousemove',
  title: 'Recording of Harlequin ladybird forms in the UK from 2004 to 2020',
  subtitle: 'Blue bars represent records from the Ladybird Survey of the UK dataset and red bars represent records from iRecord. Records for the taxon Harmonia axyridis are those for which a specific form was not recorded.',
  footer: 'Data derived from records downloaded from the NBN in June 2
})
