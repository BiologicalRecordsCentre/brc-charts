metrics Array.<Object> An array of objects, each describing a numeric property in the input data for which taxa and/or count accumulation lines should be generated on the chart. Each of the objects in the data array must be sepecified with the properties shown below. (The order is not important.)
  • prop - the name of the numeric property in the data (count properties - 'c1' or 'c2' in the example below).
  • labelTaxa - a label for the taxa accumulation metric.
  • labelCounts - a label for the counts accumulation metric.
  • key - a base key for this metric. If the options are updated using the setChartOpts API method, then if this value is set, it is used to uniquely identify the graphic elements. If not set, then the label is used.
  • ColourTaxa - optional colour to give the line for the taxa accumulation metric. Any accepted way of specifying web colours can be used. Use the special term 'fading' to successively fading shades of grey.
  • ColourCounts - optional colour to give the line for the counts accumulation metric. Any accepted way of specifying web colours can be used. Use the special term 'fading' to successively fading shades of grey.
  • styleTaxa - options style to give the line for the taxa accumulation metric. Accepted value is 'dashed' which results in a dashed line. Anything else results in a solid line.
  • styleCounts - options style to give the line for the counts accumulation metric. Accepted value is 'dashed' which results in a dashed line. Anything else results in a solid line.
data Array.<Object> Specifies an array of data objects. Each of the objects in the data array must be sepecified with the properties shown below. (The order is not important.)
  • taxon - name of a taxon.
  • week - a number between 1 and 53 indicating the week of the year.
  • c1 - a count for a given time period (can have any name).
  • c2 - a count for a given time period (can have any name). ... - there can be any number of these count columns.
