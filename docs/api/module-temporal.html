metrics Array.<Object> An array of objects, each describing a numeric property in the input data for which graphics should be generated on the chart. Each of the objects in the data array can be sepecified with the properties shown below. (The order is not important.)
  • prop - the name of the numeric property in the data (metric properties - 'c1' or 'c2' in the example below).
  • id - a unique identifier for the property. This is only necessary if metrics can be interactively added or removed on a chart. Under these circumstances, transitions and highlighting can get mixed up if each metric does not have a consistent unique id.
  • taxon - an optional property that can be used to limit the metric display to a particular taxon. This is meant to be used in conjunction with the opts.taxa value of [null] - see above.
  • periodMin a value for period, before which points should not be displayed.
  • periodMax a value for period, after which points should not be displayed.
  • label - a label for this metric. (Optional - the default label will be the property name.)
  • colour - optional colour to give the graphic for this metric. Any accepted way of specifying web colours can be used. Use the special term 'fading' to successively fading shades of grey. (Optional - default is 'blue'.)
  • opacity - optional opacity to give the graphic for this metric. (Optional - default is 1.)
  • strokewidth - optional width of line for line for this metric if displayed as a line graph. (Optional - default is 1.)
  • fill - optional colour to give the area fill on area charts for this metric. Any accepted way of specifying web colours can be used. Use the special term 'fading' to successively fading shades of grey.
  • fillOpacity - optional opacity to give the area fill on area charts for this metric. (Optional - default is 0.5.)
  • bandUpper - optional name of a numeric property in the data which indicates the upper value of a confidence band. Can only be used where chartStyle is 'line'.
  • bandLower - optional name of a numeric property in the data which indicates the lower value of a confidence band. Can only be used where chartStyle is 'line'.
  • bandFill - optional colour to use for a confidence band. Any accepted way of specifying web colours can be used. (Optional - default is 'silver'.)
  • bandStroke - optional colour to use for the uppder and lower boundaries of a confidence band. Any accepted way of specifying web colours can be used. (Optional - default is 'grey'.)
  • bandOpacity - optional opacity to give the confidence band for this metric. (Optional - default is 0.5.)
  • bandStrokeOpacity - optional opacity to give the boundaries of the confidence band for this metric. (Optional - default is 1.)
  • bandStrokewidth - optional width of line for bounary lines of the confidence band this metric if displayed as a line graph. (Optional - default is 1.)
  • points - a boolean value which indicates whether or not a point is to be displayed (over the line or bar).
  • errorBarUpper - optional name of a numeric property in the data which indicates the upper value of an error bar. Used in conjunction with the errorBarLower property.
  • errorBarLower - optional name of a numeric property in the data which indicates the lower value of an error bar. Used in conjunction with the errorBarUpper property.
data Array.<Object> Specifies an array of data objects. Each of the objects in the data array must be sepecified with the properties shown below. (The order is not important.)
  • taxon - name of a taxon.
  • period - a number indicating a week, nonth or a year.
  • c1 - a metric for a given period (can have any name).
  • c2 - a metric for a given period (can have any name). ... - there must be at least one metric column, but there can be any number of them.
dataPoints Array.<Object> Specifies an array of data objects. Each of the objects in the data array must be sepecified with the properties shown below. (The order is not important.)
  • taxon - name of a taxon. This is optional. If not specified, then data are shown regardless of selected taxon.
  • period - a number indicating a week, month or a year.
  • y - y value for a given period.
  • upper - a value for upper confidence band.
  • lower - a value for lower confidence band.
dataTrendLines Array.<Object> Specifies an array of data objects. Each of the objects in the data array must be sepecified with the properties shown below. (The order is not important.)
  • id - if the trend line is to be associated with a particular metric, then set this value to match the id property of the metric. Then the line(s) will be included in highlighting/lowlighting for the metric.
  • taxon - name of a taxon. This is optional. If not specified, then data are shown regardless of selected taxon.
  • gradient - a gradient for the line (either specify gradient & intercept or p1, p2, v1 and v2).
  • intercept - the y axis intercept value (at x = 0) for the line (either specify gradient & intercept or p1, p2, v1 and v2).
  • p1 - the lower period for the line (either specify gradient & intercept or p1, p2, v1 and v2).
  • p2 - the upper period for the line (either specify gradient & intercept or p1, p2, v1 and v2).
  • v1 - the lower value for the line (either specify gradient & intercept or p1, p2, v1 and v2).
  • v2 - the upper value for the line (either specify gradient & intercept or p1, p2, v1 and v2).
  • colour - the colour of the line the line. Any accepted way of specifying web colours can be used. (Default - red.)
  • width - the width the line the line in pixels. (Default - 1.)
  • opacity - the opacity of the line. (Default - 1.)
verticals Array.<Object> Specifies an array of data objects for showing vertical lines and bands on a chart. Each of the objects in the data array must be sepecified with the properties shown below. (The order is not important.)
  • taxon - name of a taxon. This is optional. If not specified, then data are shown regardless of selected taxon.
  • colour - the colour of the line or band. Any accepted way of specifying web colours can be used. (Default - red.)
  • start - a value to indicate the position on the x axis where the line will be drawn (or band start). For periodType of 'year' this value is specified in units of years. For periodType of 'month' or 'week', this value is specified in *days*. (See below for values than map to the first day for each month.)
  • width - the width of the band to be drawn. If absent or zero, then a line is drawn rather than a band. Specified in the same units as the 'start' value. (Default - 0.).
The numbers used for the first of the month for each month Jan to Dec are: 1, 32, 61, 92, 122, 153, 183, 214, 245, 275, 306 and 336.
taxa Array.<string> An array of taxa (names), indicating which taxa create charts for. If empty, graphs for all taxa are created. If set to an array with a single null value - [null] - then each metric must have a property called 'taxon' which specifies the taxon to be displayed for that metric. In this way, multiple taxa can be displayed on the same graph. (Default - [].)
periodType string Indicates the type of period data to be specified. Can be 'year', 'month' or 'week'. (Default - 'year'.)
minPeriod number Indicates the earliest period to use on the x axis. If left unset, the earliest period in the dataset is used. (Default - null.)
maxPeriod number Indicates the latest period to use on the x axis. If left unset, the latest period in the dataset is used. (Default - null.)
minPeriodTrans number If set, this indicates the lowest possible period. It is only useful if transitioning between datasets with different temporal ranges - its purpose is to facilitate smooth transitions of lines and bands in these cases. (Default - null.)
maxPeriodTrans number If set, this indicates the highest possible period. It is only useful if transitioning between datasets with different temporal ranges - its purpose is to facilitate smooth transitions of lines and bands in these cases. (Default - null.)
minY number Indicates the lowest value to use on the y axis. If left unset, the lowest value in the dataset is used. (Default - null.)
maxY number Indicates the highest value to use on the y axis. If left unset, the highest value in the dataset is used. (Default - null.)
minMaxY number Indicates a minumum value for the maximum value. Set to null for no minimum value. (Default is 5.)
xPadPercent number Padding to add either side of min and max period value - expressed as percentage of temporal range. Can only be used on line charts. (Default - 0.)
yPadPercent number Padding to add either side of min and max y value - expressed as percentage of y range. Can only be used on line charts. (Default - 0.)
missingValues string | number A value which indicates how gaps in temporal data are treated. Can either be the string value 'break' which leaves gaps in line charts, 'bridge' which joins the points either side of a missing value or a numeric value which is used to replace the missing value. (Default - 'break'.)
monthScaleRange Array.<number> A two value numeric array indicating which months to include on annual scales (for periodType 'month' or 'week'). (Default - [1,12].) causes trend lines to break where no value, 'bridge' which causes gaps to be bridged with straight line or a numeric value. (Default - 0.)
lineInterpolator boolean Set to the name of a d3.line.curve interpolator to curve lines. (Default - 'curveLinear'.)
metricExpression string Indicates how the metric is expressed can be '' to leave as is, or 'proportion' to express as a proportion of the total of the metric or 'normalized' to normalize the values. (Default - ''.)
composition string Indicates how to display multiple metrics. If set to empty string then the metrics values are overlaid on each other, setting to 'stack' stacks the graphics and setting to 'spread' spreads them vertically across the chart. (Default - ''.)
overrideHighlight boolean if set to true, the default styles for highlighting and lowlighting are not applied and you can provide your own CSS to do it. Target .temporal-graphic.lowlight and .temporal-graphic.highlight. (Default - false.)
spreadOverlap number a number between 0 and 1 that indicates how much overlap is permitted on graphics for different metrics on a chart of composition type 'spread'. (Default - 0.8.)
watermark Object Options for creating a watermark on the chart. (Default - {}.) The watermark object can have any of the properties shown below. (The order is not important.) The only mandatory property (if a watermark is required) is 'text'.
  • text - a string specifying the text that comprises the watermark.
  • font - a string specifying the font family. If ommitted, the default font is used.
  • fontSize - an integer specifying font size (in pixels). (Default - 20.)
  • fontWeight - a string that specifies the font weight. (Default - 'bolder'.)
  • colour - a string that specifies the text colour. Any accepted web format can be used. (Default - 'black'.)
  • opacity - a number between 0 and 1 that indicates the opacity of the text. (Default 0.2.)
  • rotation - a number between 0 and 359 that indicates the rotation of the text in degrees. (Default 0.)
