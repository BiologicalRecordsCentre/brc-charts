taxa Array.<string> An array of taxa (names), indicating which taxa create charts for. If empty, graphs for all taxa are created.
metrics Array.<Object> An array of objects, each describing a property in the input data for which a band should be generated on the chart. Each of the objects in the data array must be sepecified with the properties shown below.
  • prop - the name of the property in the data (properties - 'p1' or 'p2' in the example below).
  • label - a label for this property.
  • colour - colour to give the band for this property. Any accepted way of specifying web colours can be used.
  • svg - Optinal string defining an SVG path of an icon to use in place of a colour swatch in the legend.
The order in which the metrics are specified determines the order in which properties are drawn on the chart. Each is drawn over the previous so if you are likely to have overlapping properties, the one you want to draw on top should come last. Because this will generally be the most important, the order is reversed for the chart legend.
data Array.<Object> Specifies an array of data objects. Each of the objects in the data array must be sepecified with the properties shown below. There should only be one object per taxon. (The order is not important.)
  • taxon - name of a taxon.
  • p1 - a date band object (see below), indicating start and end weeks for the property (can have any name).
  • p2 - a date band object (see below), indicating start and end weeks for the property (can have any name). ... - there must be at leas one property column, but there can be any number of them.
The date band objects have the following structure:
  • start - a number between 1 and 365 indicating the day of the year the band starts.
  • end - a number between 1 and 365 indicating the day of the year the band ends.

(inner) setChartOpts(opts)

This function is exposed as a method on the API returned from the phen2 function. Set's the value of the chart data, title, subtitle and/or footer. If an element is missing from the options object, it's value is not changed.
