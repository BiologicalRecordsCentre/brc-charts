taxa Array.<string> An array of taxa (names), indicating which taxa create charts for. If empty, graphs for all taxa are created.
metrics Array.<Object> An array of objects, each describing a property in the input data for which a band should be generated on the chart. Each of the objects in the metrics array must be sepecified with the properties shown below.
  • prop - the name of the property in the data (properties - 'p1' or 'p2' in the example below).
  • label - a label for this property.
  • colour - colour to give the band for this property. Any accepted way of specifying web colours can be used.
  • opacity - Optional opacity to give the band for this property. A value between 0 and 1 - default is 1.
  • svg - Optional string defining an SVG path of an icon to use in place of a colour swatch in the legend.
  • svgScale - Optional number defining a scaling factor to apply to SVG icon (relative to others) - default is 1.
  • legendOrder - Optional number used to sort the legend items. Low to high = left to right. If supplied, it should be supplied for all metrics items otherwise results undefined. If not defined, default is to reverse the order. Numbering should start at one and increase in increments of 1. For split bands it is also used to position the bands from top to bottom
The order in which the metrics are specified determines the order in which properties are drawn on the chart. Each is drawn over the previous so for overlapping properties (split = false), the one you want to draw on top should come last.
data Array.<Object> Specifies an array of data objects. Each of the objects in the data array must be sepecified with the properties shown below. There should only be one object per taxon. (The order is not important.)
  • taxon - name of a taxon.
  • p1 - an array of date band objects (see below), indicating start and end weeks for the property (can have any name).
  • p2 - an array of date band objects (see below), indicating start and end weeks for the property (can have any name). ... - there must be at leas one property column, but there can be any number of them.
The objects in each data band array have the following structure:
  • start - a number between 1 and 365 indicating the day of the year the band starts.
  • end - a number between 1 and 365 indicating the day of the year the band ends.
